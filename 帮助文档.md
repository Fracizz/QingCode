# QingCode 配置帮助

本文说明 QingCode 的界面语言包、项目与临时项目，以及项目自定义设置。语言包随应用发布；普通项目会写入本机项目列表；临时项目仅存在于当前运行会话。项目设置保存在项目目录中，便于和项目一起管理。

## 选择界面语言

1. 打开左侧的“设置”。
2. 展开“语言”。
3. 选择“简体中文”或 “English”。

选择结果会保存在本机；下次启动 QingCode 时会自动恢复。

## 临时项目

「新建临时项目」用于在不选择已有文件夹的情况下，立刻获得一个空白工作区。适合快速记笔记、试命令、或临时改几份文件，而不用先把目录加入项目列表。

### 如何创建

1. 点击顶栏项目芯片右侧的 **「+」** 按钮（悬停提示为「新建临时项目」）；没有任何项目时，顶栏也会显示 **「新建临时项目」** 文字按钮。
2. 当前没有任何项目时，点击终端面板的 **「新建终端」** 也会自动创建一个临时项目，以便终端有工作目录。

创建成功后，顶栏会出现名为「临时项目」「临时项目 2」…… 的芯片，并自动切换为当前项目。

### 与普通项目的区别

| | 普通项目（添加目录） | 临时项目 |
| --- | --- | --- |
| 来源 | 选择本机已有文件夹 | 在系统临时目录下新建空文件夹 |
| 项目列表 | 写入本机项目数据库，重启后仍在 | **不写入**项目数据库，**重启后从列表消失** |
| 项目管理 | 可在「项目管理」中查看、排序、隐藏 | **不会**出现在「项目管理」中 |
| 命名工作区 | 可纳入命名工作区快照 | **不会**写入命名工作区快照 |
| 工作区信任 | 首次使用需选择信任或受限 | 自动视为已信任，可直接编辑与开终端 |
| 本次会话 | 可编辑、终端、搜索、保存文件 | 同样可用，行为与普通项目一致 |

### 文件保存在哪里

临时项目的根目录位于系统临时文件夹下（名称形如 `qingcode-empty-<随机 id>`）。你在其中新建或保存的文件会正常写入磁盘；**关闭 QingCode 或重启后，应用不再显示该项目**，但临时文件夹本身不会由 QingCode 自动删除，可能仍留在系统临时目录中，直至操作系统清理。

### 移除与长期保留

- 从顶栏关闭临时项目芯片：只从**当前会话**的项目列表中移除，不会删除磁盘上的临时文件夹。
- 若希望长期保留其中的文件：在文件管理器中复制或移动到正式目录，再用 **「添加项目」** 把该目录加入项目列表；或先在临时项目中编辑，完成后手动备份内容。

## 快捷键与搜索

打开“设置”后，展开“快捷键”即可修改常用操作的按键组合。当前默认值为：

- `Ctrl+Shift+F`：打开搜索面板，默认在当前项目中搜索文件和文件内容，并聚焦搜索输入框；需要时可手动切换到“全部项目”。
- `Ctrl+\``：显示或隐藏终端面板。
- `Ctrl+,`：打开设置面板。

编辑器保留（设置页只读展示，不可改绑）：

- `Ctrl+S`：保存当前文件。
- `Ctrl+Shift+C`：复制文件完整路径。
- `Alt+C`：复制为文件引用（含行号范围）。
- `Shift+Alt+F`：格式化当前文件（Prettier / rustfmt，需本机已安装）。

点击某个可编辑快捷键输入框后，直接按新的组合键即可保存；点击“恢复默认快捷键”可还原以上可自定义默认值。

## 国际化 JSON 语言包

内置语言包位于 `src/locales/`：

- `zh-CN.json`：简体中文语言包。
- `en.json`：英文语言包。

每个语言包使用相同的 JSON 结构：

```json
{
  "locale": "en",
  "label": "English",
  "messages": {
    "设置": "Settings",
    "保存": "Save"
  }
}
```

`messages` 的键是界面中的原始中文文案，值是目标语言文案。界面通过 `t('设置')` 读取翻译；如果某个键没有配置，会继续显示原始文案，不会造成界面报错。

新增一种语言时，添加对应的 JSON 文件，并在 `src/lib/i18n.ts` 中注册语言包和语言代码。设置页会自动从已注册的语言包读取名称。

## 自定义设置 JSON（全局 / 项目）

QingCode 支持两级设置，语义接近 VS Code：

| 作用域 | 位置 | 说明 |
| --- | --- | --- |
| 全局 | 开发环境：`.dev/default-settings.json`；正式版：应用数据目录下的 `default-settings.json` | 对所有项目生效（User Settings） |
| 项目 | 项目根目录 `.qingcode/project-settings.json` | 仅当前项目，并覆盖同名全局项（Workspace Settings） |

首次保存时会自动创建目录和文件。默认模板包含常见的 VS Code 风格键（如 `editor.*`、`files.*`、`search.*`、`terminal.integrated.*`），以及预留的 `custom` 对象。

```json
{
  "version": 1,
  "editor.fontSize": 14,
  "editor.tabSize": 4,
  "files.autoSave": "off",
  "files.exclude": {
    "**/node_modules": true,
    "**/dist": true
  },
  "search.exclude": {
    "**/node_modules": true,
    "**/target": true
  },
  "custom": {
    "apiBaseUrl": "http://localhost:3000"
  }
}
```

多数常用键已生效。JSON 注释中标为「不计划」的键会保存但不会改变行为。

**已生效（摘录）：**

- 编辑器：`editor.fontSize`、`editor.tabSize`、`editor.insertSpaces`、`editor.detectIndentation`、`editor.wordWrap`、`editor.lineNumbers`、`editor.renderWhitespace`、`editor.formatOnSave`、`editor.formatOnPaste`、`editor.bracketPairColorization.enabled`、`editor.guides.bracketPairs`
- 文件：`files.autoSave`、`files.autoSaveDelay`、`files.eol`、`files.encoding`（默认 `auto`：BOM → 严格 UTF-8 → GB18030 兼容兀底；也可固定为 `utf8` / `utf8bom` / `gbk` / `gb18030`；大文件只读分片仍按 UTF-8）、`files.trimTrailingWhitespace`、`files.insertFinalNewline`、`files.exclude`（资源管理器隐藏）、`files.maxSizeForEdit`（大文件分级上限）
- 搜索：`search.exclude`、`search.useIgnoreFiles`、`search.followSymlinks`（默认 `false`）
- 资源管理器：`explorer.excludeGitIgnore`（按 `.gitignore` 等隐藏）
- 终端：`terminal.integrated.scrollback`（含跨重启输出缓冲）、`terminal.integrated.cursorBlinking`
- 全局：`qingcode.projects`、`qingcode.projects.syncOnStartup`

**不计划（预留键）：** `editor.minimap.enabled`、`editor.linkedEditing`。

`custom` 仍可用于完全自定义字段。

已打开的文件会在状态栏显示实际使用的编码。左侧下拉框是「转换编码，保存后生效」；右侧「重新打开」会丢弃未保存修改后按选定编码重读磁盘文件，两者是独立操作。GBK 是 GB18030 的子集，不能仅凭字节可靠地区分时，自动模式会选择兼容的 GB18030。

## 在界面中编辑

设置界面复刻 VS Code Settings Editor，在主编辑区打开：

1. 点击活动栏「设置」，或按 `Ctrl+,`。
2. 顶部切换「用户 / 工作区」；搜索框可过滤设置项。
3. 左侧目录跳转到分类；右侧为设置列表（标题、说明、控件；蓝条表示相对默认已修改）。
4. 右上角 JSON 图标，或目录「打开设置 JSON」：在编辑器中打开对应 `default-settings.json` / `project-settings.json`。
5. JSON 编辑后按 `Ctrl+S` 保存。

未选择项目时「工作区」不可用；多数界面设置仅用户作用域可改，工作区主要用于 `project-settings.json`。

## 与运行配置的关系

项目运行配置使用另一个文件：

```text
.qingcode/run.json
```

编辑 `default-settings.json` / `project-settings.json` 不会修改运行配置；编辑 `run.json` 也不会覆盖自定义设置。

## 给后续功能开发者

可通过 `src/lib/projectSettings.ts` 使用以下接口：

- `loadGlobalSettings()` / `saveGlobalSettings(settings)`
- `loadProjectSettings(project)` / `saveProjectSettings(project, settings)`
- `DEFAULT_SETTINGS`：默认模板
- `PROJECT_SETTINGS_RELATIVE_PATH`：项目配置相对路径
- `SettingsFile`：文件结构类型

合并配置时建议：项目值覆盖全局同名键；对象类设置（如 `files.exclude`）可按需做浅/深合并。保留 `custom` 中未知内容，避免覆盖用户数据。
