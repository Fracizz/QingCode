# QingCode 配置帮助

本文说明 QingCode 的界面语言包和项目自定义设置。两类配置互不影响：语言包随应用发布，项目设置保存在项目目录中，便于和项目一起管理。

## 选择界面语言

1. 打开左侧的“设置”。
2. 展开“语言”。
3. 选择“简体中文”或 “English”。

选择结果会保存在本机；下次启动 QingCode 时会自动恢复。

## 快捷键与全项目搜索

打开“设置”后，展开“快捷键”即可修改常用操作的按键组合。当前默认值为：

- `Ctrl+Shift+F`：在全部可用项目中搜索文件和文件内容；会自动切换到“搜索 → 全部项目”，并聚焦搜索输入框。
- `Ctrl+\``：显示或隐藏终端面板。
- `Ctrl+,`：打开设置面板。

点击某个快捷键输入框后，直接按新的组合键即可保存。保存、复制文件路径、文件引用以及开发工具相关按键由编辑器保留，不能在这里覆盖；点击“恢复默认快捷键”可还原以上默认值。

## 国际化 JSON 语言包

内置语言包位于 `src/locales/`：

- `zh-CN.json`：简体中文语言包。
- `en.json`：英文语言包。

每个语言包使用相同的 JSON 结构：

```json
{
  "locale": "en",
  "label": "English",
  "messages": {
    "设置": "Settings",
    "保存": "Save"
  }
}
```

`messages` 的键是界面中的原始中文文案，值是目标语言文案。界面通过 `t('设置')` 读取翻译；如果某个键没有配置，会继续显示原始文案，不会造成界面报错。

新增一种语言时，添加对应的 JSON 文件，并在 `src/lib/i18n.ts` 中注册语言包和语言代码。设置页会自动从已注册的语言包读取名称。

## 项目自定义设置 JSON

项目级配置文件固定在项目根目录：

```text
.qingcode/settings.json
```

首次保存时，QingCode 会自动创建 `.qingcode` 目录和配置文件。默认内容如下：

```json
{
  "version": 1,
  "custom": {}
}
```

把未来项目专属配置写在 `custom` 中即可。例如：

```json
{
  "version": 1,
  "custom": {
    "apiBaseUrl": "http://localhost:3000",
    "showExperimentalTools": true,
    "deploy": {
      "environment": "test"
    }
  }
}
```

目前 `custom` 是预留的自由 JSON 对象：QingCode 会保留并读写其中内容，但不会自动把这些字段当作内置功能开关。后续项目功能可按需读取它们。

## 在界面中编辑项目设置

1. 先在标题栏选择一个项目。
2. 打开“设置”。
3. 展开“项目自定义设置”。
4. 修改 JSON 后点击“保存”。

保存前会检查：

- 最外层必须是 JSON 对象；
- `version` 当前必须为数字 `1`；
- `custom` 必须是 JSON 对象。

未选择项目时不能保存项目设置。点击“重新加载”会从当前项目的 `.qingcode/settings.json` 重新读取内容；文件尚不存在时会显示默认模板。

## 与运行配置的关系

项目运行配置使用另一个文件：

```text
.qingcode/run.json
```

编辑 `settings.json` 不会修改运行配置；编辑 `run.json` 也不会覆盖项目自定义设置。

## 给后续功能开发者

可通过 `src/lib/projectSettings.ts` 使用以下接口：

- `loadProjectSettings(project)`：读取并解析当前项目设置；
- `saveProjectSettings(project, settings)`：保存设置；
- `PROJECT_SETTINGS_RELATIVE_PATH`：配置文件相对路径；
- `ProjectSettingsFile`：当前文件结构类型。

新增内置字段时，请保留 `custom` 中未知内容，避免覆盖用户已保存的项目自定义数据。
