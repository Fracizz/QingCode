## 快速开始

1. **添加项目**：点击顶栏右侧的 **「+」** 按钮，选择本地文件夹加入项目列表。
2. **切换项目**：点击顶栏项目 chip 即可切换，编辑器和终端状态自动保留。
3. **打开文件**：左侧文件树中点击文件，在编辑器中打开。
4. **打开终端**：按 `` Ctrl+` `` 显示终端面板，终端默认在项目根目录启动。
5. **保存文件**：按 `Ctrl+S` 保存当前文件。

## 界面语言

1. 点击左侧活动栏的 **「设置」** 图标。
2. 展开 **「语言」** 分类。
3. 在 **「显示语言」** 中选择内置语言（简体中文 / English）或已加载的自定义语言包。

> 语言选择保存在本机，下次启动自动恢复。

### 自定义语言包（用户侧）

可在不重新编译的情况下添加或覆盖界面文案：

1. 在设置 → **语言** 中点击 **「打开语言包目录」**（或手动打开应用数据目录下的 `locales` 文件夹）。
   - 发行版：`%APPDATA%\com.qingcode.app\locales\`
   - 开发版：项目根目录 `.dev/locales/`
2. 参考目录中的 `_example.json`，新建例如 `ja.json`：

```json
{
  "locale": "ja",
  "label": "日本語",
  "messages": {
    "设置": "設定",
    "语言": "言語",
    "显示语言": "表示言語"
  }
}
```

3. 回到设置点击 **「刷新语言包」**，或重启应用；新语言会出现在「显示语言」列表中。
4. 未翻译的文案会回退显示中文源键（与内置英文包未覆盖项的行为一致）。
5. 若文件名对应已有内置语言（如 `en.json`），其中的 `messages` 会**覆盖/补充**内置包，便于局部改词。

规则：

- 仅加载 `locales/*.json`；以 `_` 开头的文件（如 `_example.json`）不会作为可选语言出现。
- `locale` 使用 BCP 47 风格代码（如 `en`、`zh-CN`、`pt-BR`）。
- `messages` 的键为界面中文源文案，值为目标语言译文。

## 项目管理

### 添加项目

- 点击顶栏右侧 **「+」** 按钮，选择本地文件夹。
- 添加后项目以 chip 形式显示在顶栏，单击即可切换。

### 临时项目

适合快速记笔记、试命令，无需选择已有文件夹：

- 点击顶栏 **「+」** 按钮旁的 **「新建临时项目」**。
- 或在没有项目时，点击终端面板的 **「新建终端」** 自动创建。

| 对比项 | 普通项目 | 临时项目 |
|--------|----------|----------|
| 来源 | 选择本地已有文件夹 | 在系统临时目录新建空文件夹 |
| 持久性 | 写入项目数据库，重启后仍在 | 不写入数据库，重启后消失 |
| 项目管理 | 可在「项目管理」中查看、排序 | 不出现在「项目管理」中 |
| 信任状态 | 首次需手动选择信任 | 自动视为已信任 |

> 临时项目的文件正常写入磁盘，关闭应用后项目入口消失，但文件仍保留在系统临时目录中。

### 移除项目

- 点击项目 chip 上的 `×` 移除（仅从列表移除，不删除磁盘文件）。
- 在「项目管理」中可批量管理、排序或隐藏项目。

## 编辑器

### 基本操作

- **打开文件**：文件树中单击选中，双击打开；选中后按 `Enter` 也可打开。
- **多选与整理**：`Ctrl`/`Shift` 多选；拖放或 `Ctrl+X`/`C`/`V` 剪切、复制、粘贴；`F2` 行内重命名。
- **切换标签**：点击标签栏，或使用 `Ctrl+Tab`。
- **关闭标签**：点击标签上的 `×`，或中键点击标签。
- **保存文件**：`Ctrl+S`。
- **未保存标记**：标签上有黄色圆点提示，关闭前会提示保存。

### 编码支持

编辑器自动识别文件编码：

- **自动检测**（默认）：按 BOM → UTF-8 → GB18030 顺序检测。
- **手动切换**：状态栏显示当前编码，点击可重新按指定编码打开或转换编码保存。
- 支持 `utf8` / `utf8bom` / `utf16le` / `utf16be` / `gbk` / `gb18030`（`auto` 可识别带 BOM 的 UTF-16；无 BOM 需手动指定）。

### 大文件处理

- 超过一定大小的文件会自动降级为只读查看，避免卡顿。
- 超大文件仅提供纯文本查看模式。

### 格式化

- **手动格式化**：`Shift+Alt+F`，调用本机已安装的 Prettier / rustfmt / shfmt / ruff·black / gofmt。
- **保存时自动格式化**：在设置中开启 `editor.formatOnSave`。
- **粘贴后自动格式化**：在设置中开启 `editor.formatOnPaste`。

### 小地图

- 编辑区右侧显示代码缩略图（默认开启），便于感知结构与快速跳转。
- **开关**：设置 → 文本编辑器 → 小地图（`editor.minimap.enabled`），或 `Ctrl+Shift+G` / 命令面板「切换小地图」。
- **交互**：点击或拖拽跳转；悬停可 Quick View 附近源码；**左缘**拖拽调宽（与侧边栏同款 `PanelResizer`：`ew-resize`、竖线 + 三点 grip）；右键可切换「划过滚动条显示小地图」「隐藏编辑器滚动条」。
- **体积**：≤1MB 语法着色；1–5MB 密度条；大于 5MB 自动隐藏。Diff / 只读大文件查看器不显示。

## 终端

### 打开终端

- `` Ctrl+` ``：显示或隐藏终端面板。
- 点击终端面板顶部的 **「+」** 新建终端标签。
- 终端默认在项目根目录启动。

### 终端配置

可为终端配置默认启动命令：

1. 打开 **「设置」** → **「功能」** → **「终端配置」**。
2. 添加配置文件（名称 + 启动命令）。
3. 新建终端时右键 **「+」** 按钮选择配置。

> 常用场景：一键进入 AI CLI（如 `opencode`）或项目特定的脚本环境。

### 运行配置与终端的关系

- **运行配置**：管「项目服务怎么起」，一键启动多个服务任务。
- **终端配置**：管「这个壳默认跑什么」，控制单个终端的初始环境。

## 搜索

### 打开搜索

- `Ctrl+Shift+F`：打开搜索面板。
- 默认在当前项目中搜索，可手动切换到「全部项目」。

### 搜索范围

- **文件名搜索**：快速定位文件。
- **文件内容搜索**：在项目内搜索文本内容。
- 支持按 `.gitignore` 等 ignore 文件过滤（可关闭）。

## Git 源代码管理

### 查看更改

1. 点击左侧活动栏的 **「源代码管理」** 图标（显示当前 Git 变更数）。
2. 查看修改列表、分支信息。
3. 点击文件可查看与 HEAD 的差异对比。

### 支持场景

- 中文、空格、重命名路径的文件差异。
- 未跟踪、已修改、已暂存、已删除等状态的文件。

> 提交与推送建议在你熟悉的 Git 工具或 AI 助手中完成。

## 运行配置

为项目配置一组「运行配置」，一键启动多个服务：

### 创建运行配置

1. 打开 **「设置」** → **「运行配置」**。
2. 点击 **「添加配置」**，输入配置名称。
3. 在配置下添加任务（命令 / 脚本 / ps1 / bat / sh）。

### 启动运行配置

- 点击配置卡片上的 **「启动」** 按钮。
- 每个任务自动开一个终端，适合同时拉起 API、前端、Worker 等。
- 点击 **「停止」** 可停止整组配置的所有任务。

### 配置保存位置

运行配置保存在项目根目录的 `.qingcode/run.json` 中，便于与项目一起管理。

## 设置

### 设置层级

| 作用域 | 位置 | 说明 |
|--------|------|------|
| 用户设置 | 应用数据目录 `default-settings.json` | 对所有项目生效 |
| 工作区设置 | 项目根目录 `.qingcode/project-settings.json` | 仅当前项目，覆盖同名全局项 |

### 文件格式（JSON5）

`default-settings.json` 与 `.qingcode/project-settings.json` 均为 **JSON5**（不是严格 JSON）：

- 支持 `//`、`/* */` 注释、尾逗号、无引号键名等。
- 默认模板已为各配置项写好说明；模板注释中写明 **【不得删除注释】**，请保留文件头与各项说明。
- 手工编辑时请继续用 JSON5 写法（应用按 JSON5 解析）。
- 发行版路径示例：`%APPDATA%\com.qingcode.app\default-settings.json`；开发版在项目 `.dev/default-settings.json`。

### 常用设置项

- **编辑器**：`editor.fontSize`、`editor.tabSize`、`editor.wordWrap`、`editor.lineNumbers`
- **文件**：`files.autoSave`、`files.exclude`、`files.encoding`
- **搜索**：`search.exclude`、`search.useIgnoreFiles`
- **终端**：`terminal.integrated.scrollback`、`terminal.integrated.cursorBlinking`

### 编辑设置

1. 按 `Ctrl+,` 打开设置面板。
2. 顶部切换「用户 / 工作区」。
3. 搜索框过滤设置项，或直接点击右上角 JSON 图标编辑原始 **JSON5**（可写注释）。

## 快捷键

### 常用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+S` | 保存当前文件 |
| `Ctrl+Shift+F` | 打开搜索面板 |
| `` Ctrl+` `` | 显示/隐藏终端面板 |
| `Ctrl+,` | 打开设置面板 |
| `Ctrl+Shift+C` | 复制文件完整路径 |
| `Alt+C` | 复制为文件引用（含行号范围） |
| `Shift+Alt+F` | 格式化当前文件 |

### 自定义快捷键

1. 打开 **「设置」** → **「快捷键」**。
2. 点击要修改的快捷键输入框。
3. 直接按新的组合键即可保存。
4. 点击 **「恢复默认快捷键」** 还原默认值。

> 编辑器保留的快捷键（如 `Ctrl+S`、`Ctrl+Shift+C`）只读展示，不可修改。

## 检查更新

### 自动检查

- 正式版启动约 3 秒后自动查询最新 Release。
- 发现新版本时提示打开下载页（不会自动安装）。

### 手动检查

- **设置** → **「功能」** → **「检查更新」** 按钮。
- 或点击状态栏右下角的版本号。

### 关闭自动检查

- **设置** → **「功能」** → **「启动时自动检查更新」** 选 **「关闭」**。

## 常见问题

### 项目无法编辑/终端无法使用

未信任项目会限制编辑、终端和脚本运行能力。点击提示中的 **「信任」** 即可。

### 文件编码显示乱码

- 状态栏会显示当前检测到的编码。
- 点击编码可选择「重新打开」（按新编码读取，不保存）或「转换编码保存」。

### 外部工具修改了正在编辑的文件

- 干净标签（未修改）会自动重载。
- 脏标签（有未保存修改）会提示比较差异、重载或保留本地修改。

### 临时项目文件如何长期保留

- 在文件管理器中复制临时项目文件夹到正式目录。
- 然后用 **「添加项目」** 把该目录加入项目列表。

## 语言包开发（发行内置）

用户侧自定义见上文 [自定义语言包（用户侧）](#自定义语言包用户侧)。若要将语言打进安装包：

1. 在 `src/locales/` 添加 JSON（格式同用户包，参考 `zh-CN.json` / `en.json`）。
2. 在 `src/lib/i18n.ts` 的 `builtinPackages` 中注册。
3. 重新构建后，该语言作为内置项出现在设置中。
