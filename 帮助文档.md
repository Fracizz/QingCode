# QingCode 配置帮助

本文说明 QingCode 的界面语言包和项目自定义设置。两类配置互不影响：语言包随应用发布，项目设置保存在项目目录中，便于和项目一起管理。

## 选择界面语言

1. 打开左侧的“设置”。
2. 展开“语言”。
3. 选择“简体中文”或 “English”。

选择结果会保存在本机；下次启动 QingCode 时会自动恢复。

## 快捷键与搜索

打开“设置”后，展开“快捷键”即可修改常用操作的按键组合。当前默认值为：

- `Ctrl+Shift+F`：打开搜索面板，默认在当前项目中搜索文件和文件内容，并聚焦搜索输入框；需要时可手动切换到“全部项目”。
- `Ctrl+\``：显示或隐藏终端面板。
- `Ctrl+,`：打开设置面板。

点击某个快捷键输入框后，直接按新的组合键即可保存。保存、复制文件路径、文件引用以及开发工具相关按键由编辑器保留，不能在这里覆盖；点击“恢复默认快捷键”可还原以上默认值。

## 国际化 JSON 语言包

内置语言包位于 `src/locales/`：

- `zh-CN.json`：简体中文语言包。
- `en.json`：英文语言包。

每个语言包使用相同的 JSON 结构：

```json
{
  "locale": "en",
  "label": "English",
  "messages": {
    "设置": "Settings",
    "保存": "Save"
  }
}
```

`messages` 的键是界面中的原始中文文案，值是目标语言文案。界面通过 `t('设置')` 读取翻译；如果某个键没有配置，会继续显示原始文案，不会造成界面报错。

新增一种语言时，添加对应的 JSON 文件，并在 `src/lib/i18n.ts` 中注册语言包和语言代码。设置页会自动从已注册的语言包读取名称。

## 自定义设置 JSON（全局 / 项目）

QingCode 支持两级设置，语义接近 VS Code：

| 作用域 | 位置 | 说明 |
| --- | --- | --- |
| 全局 | 开发环境：`.dev/default-settings.json`；正式版：应用数据目录下的 `default-settings.json` | 对所有项目生效（User Settings） |
| 项目 | 项目根目录 `.qingcode/project-settings.json` | 仅当前项目，并覆盖同名全局项（Workspace Settings） |

首次保存时会自动创建目录和文件。默认模板包含常见的 VS Code 风格键（如 `editor.*`、`files.*`、`search.*`、`terminal.integrated.*`），以及预留的 `custom` 对象。

```json
{
  "version": 1,
  "editor.fontSize": 14,
  "editor.tabSize": 4,
  "files.autoSave": "off",
  "files.exclude": {
    "**/node_modules": true,
    "**/dist": true
  },
  "search.exclude": {
    "**/node_modules": true,
    "**/target": true
  },
  "custom": {
    "apiBaseUrl": "http://localhost:3000"
  }
}
```

目前多数键为预留配置：QingCode 会完整读写并保留它们，但不会立刻全部映射到内置界面控件。后续功能可按需读取。`custom` 仍可用于完全自定义字段。

## 在界面中编辑

设置界面复刻 VS Code Settings Editor，在主编辑区打开：

1. 点击活动栏「设置」，或按 `Ctrl+,`。
2. 顶部切换「用户 / 工作区」；搜索框可过滤设置项。
3. 左侧目录跳转到分类；右侧为设置列表（标题、说明、控件；蓝条表示相对默认已修改）。
4. 右上角 JSON 图标，或目录「打开设置 JSON」：在编辑器中打开对应 `default-settings.json` / `project-settings.json`。
5. JSON 编辑后按 `Ctrl+S` 保存。

未选择项目时「工作区」不可用；多数界面设置仅用户作用域可改，工作区主要用于 `project-settings.json`。

## 与运行配置的关系

项目运行配置使用另一个文件：

```text
.qingcode/run.json
```

编辑 `default-settings.json` / `project-settings.json` 不会修改运行配置；编辑 `run.json` 也不会覆盖自定义设置。

## 给后续功能开发者

可通过 `src/lib/projectSettings.ts` 使用以下接口：

- `loadGlobalSettings()` / `saveGlobalSettings(settings)`
- `loadProjectSettings(project)` / `saveProjectSettings(project, settings)`
- `DEFAULT_SETTINGS`：默认模板
- `PROJECT_SETTINGS_RELATIVE_PATH`：项目配置相对路径
- `SettingsFile`：文件结构类型

合并配置时建议：项目值覆盖全局同名键；对象类设置（如 `files.exclude`）可按需做浅/深合并。保留 `custom` 中未知内容，避免覆盖用户数据。
